Dealer Bot

ESP32 Stepper motors Servo control SolidWorks

An automated card-dealing robot designed to dispense playing cards with repeatable feeding, controlled trajectory, and reliable player-to-player distribution.

System goal

This project focuses on integrating a moving mechanical assembly with microcontroller-driven logic. The core challenge is to separate a single card from a stack, move it into a controlled eject position, and direct it toward the correct player location.

The build combines mechanical design, rapid prototyping, motor control, web-based interaction, and iterative testing. The current system is still being refined, with emphasis on reliability, repeatability, and smooth actuation.

Subsystems

Mechanical and controls architecture

Mechanical dispensing

Engineered a friction-based roller and feed concept to separate and deal single cards from a standard deck while reducing jams. The chassis and brackets were modeled in SolidWorks and 3D printed.

Motion control

Used stepper motors, servo mechanisms, and controlled DC motor output to manage hopper motion, pusher motion, base rotation, and card ejection.

Embedded logic

Built microcontroller logic for commands, homing routines, calibration, game flow, and repeatable card-dealing sequences across multiple player positions.

Key engineering challenges

  • Separating one card at a time without double-feeding.
  • Maintaining repeatable motion across different deal distances.
  • Coordinating mechanical tolerances with software timing.
  • Designing a control system that is easy to calibrate and test.

Current direction

The next phase is focused on improving robustness through consistent motor drive logic, cleaner homing behavior, and a more polished user interface for game setup and calibration.

The project has become a strong test bed for mechanical iteration, embedded controls, and practical debugging under real hardware constraints.
